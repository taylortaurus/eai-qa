什么是 Stage Spec QA？

Stage Spec QA 是一个基于领域代码片段的测试自动化平台。通过代码片段约束测试步骤，确保测试执行的确定性和一致性。

领域驱动

通过领域代码片段约束测试步骤，确保测试执行的确定性和一致性

协议化

标准化 workflow 执行协议，固化关键步骤，减少执行不确定性

可扩展

支持通过插件集成拓展测试能力，适应不同业务场景

核心能力

用例解析

解析测试用例文档，提取测试步骤和预期结果

四层感知

领域代码片段、前端框架 API、无障碍树、视觉理解

自动执行

支持登录、导航、表单操作等复杂测试场景

报告生成

自动生成测试报告，展示执行结果和通过率

脚本落袋

通过的用例生成标准 Playwright 脚本

领域插件

支持地图选择等业务个性化场景

方案对比

Stage Spec QA

  • 领域代码片段保障确定性
  • 协议化执行，效率与精度平衡
  • 四层感知策略，适应不同页面
  • 固化 workflow，执行稳定
  • 业务场景可扩展
  • 标准 Playwright 脚本产出

纯编码 Playwright

  • 需要手动编写和维护脚本
  • 页面结构变更导致脚本失效
  • 开发成本高，迭代慢
  • 执行速度快，稳定性高
  • 调试方便，可控性强

纯视觉方案

  • 完全依赖视觉推理，执行速度慢
  • 页面样式变更影响识别
  • Token 消耗大，成本高
  • 无需了解页面结构
  • 适合复杂验证码等场景

MCP 通用方案

  • 通用方案，缺乏领域约束
  • 复杂场景需要大量 Prompt 工程
  • 执行结果不确定性高
  • 开箱即用，上手简单
  • 社区活跃，生态丰富
