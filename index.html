什么是 Stage Spec QA？

Stage Spec QA 是一个基于 Agent Skills 和领域代码片段的智能测试自动化平台。通过领域代码片段约束测试步骤，结合 AI 理解能力，实现高效、稳定的测试自动化。

🎯

领域驱动

通过领域代码片段约束测试步骤，确保测试执行的幂等性和一致性

🤖

AI 增强

利用 Claude Code 等大语言模型理解用例语义，智能编排测试流程

🔧

协议化

标准化 workflow 执行协议，固化关键步骤，减少 AI 推理不确定性

核心能力

📝

用例解析

智能理解测试用例文档，自动解析测试步骤、预期结果和测试数据

🔍

四层感知

领域代码片段 → 前端框架 API → 无障碍树 → AI 视觉理解，精确建模页面

⚡

自动执行

AI 推理驱动的测试执行，支持登录、导航、表单操作等复杂场景

📊

报告生成

自动生成 HTML 测试报告，支持用例通过率、执行时间等指标展示

💾

脚本落袋

测试通过后自动生成标准 Playwright 脚本，支持历史用例增量更新

🎨

领域插件

支持地图选择等业务个性化场景，通过插件集成拓展测试能力

对比其他方案

✨ Stage Spec QA

  • 领域代码片段保障幂等性
  • AI 理解 + 协议化执行，平衡效率与精度
  • 四层感知策略，适应不同页面类型
  • 固化 workflow，对冲模型不确定性
  • 业务场景可扩展，支持领域插件
  • 标准 Playwright 脚本产出

纯编码 Playwright

  • 需要手动编写和维护脚本
  • 页面结构变更导致脚本失效
  • 开发成本高，迭代慢
  • 执行速度快，稳定性高
  • 调试方便，可控性强

纯视觉 Midscene

  • 完全依赖视觉推理，执行速度慢
  • 页面样式变更影响识别
  • Token 消耗大，成本高
  • 无需了解页面结构
  • 适合复杂验证码等场景

Playwright MCP

  • 通用方案，缺乏领域约束
  • 复杂场景需要大量 Prompt 工程
  • 执行结果不确定性高
  • 开箱即用，上手简单
  • 社区活跃，生态丰富

# 执行测试用例

  --case-file test-cases/005-投资项目新增-测试用例.md \

  --snippet-args '{"login":{"role":"建设单位"}}' \
  --headed
