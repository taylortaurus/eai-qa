领域测试自动化解决方案

什么是 EAI-QA？

EAI-QA 是一个基于领域代码片段的测试自动化平台。通过代码片段约束测试步骤，确保测试执行的确定性和一致性。

领域驱动

通过领域代码片段约束测试步骤，确保测试执行的确定性和一致性

协议化

标准化 workflow 执行协议，固化关键步骤，减少执行不确定性

可扩展

支持通过插件集成拓展测试能力，适应不同业务场景

核心能力

用例解析

解析测试用例文档，提取测试步骤和预期结果

四层感知

领域代码片段、前端框架 API、无障碍树、视觉理解

自动执行

支持登录、导航、表单操作等复杂测试场景

报告生成

自动生成测试报告，展示执行结果和通过率

脚本落袋

通过的用例生成标准 Playwright 脚本

领域插件

支持地图选择等业务个性化场景

方案对比

EAI-QA

  • 领域代码片段保障确定性
  • 协议化执行，效率与精度平衡
  • 四层感知策略，适应不同页面
  • 固化 workflow，执行稳定
  • 业务场景可扩展
  • 标准 Playwright 脚本产出

纯编码 Playwright

  • 需要手动编写和维护脚本
  • 页面结构变更导致脚本失效
  • 开发成本高，迭代慢
  • 执行速度快，稳定性高
  • 调试方便，可控性强

纯视觉方案

  • 完全依赖视觉推理，执行速度慢
  • 页面样式变更影响识别
  • Token 消耗大，成本高
  • 无需了解页面结构
  • 适合复杂验证码等场景

MCP 通用方案

  • 通用方案，缺乏领域约束
  • 复杂场景需要大量 Prompt 工程
  • 执行结果不确定性高
  • 开箱即用，上手简单
  • 社区活跃，生态丰富

工作流程

1
初始化环境
close-all，读取 .env-stage 配置
2
读取用例文档
解析测试用例内容，提取测试步骤和预期结果
3
理解用例语义
AI 推理，编排执行规划
4
页面感知
四层感知策略，逐层尝试
第1层：领域代码片段 第2层：前端框架 API 第3层：无障碍树 AT 第4层：AI 视觉（规划中）
5
执行动作
领域能力 / API / 固化脚本
6
验证结果
检查执行结果
7
脚本落袋
生成 Playwright 脚本（通过后）
8
产出报告
result.json, HTML 报告

完整流程总结

  • 初始化 → 读取用例 → 理解语义 → 页面感知 → 执行动作 → 验证结果
  • 通过后：脚本落袋 → 产出报告
  • 失败后：回退兜底机制

快速开始

# 初始化环境
stage-cli --init-env

# 执行测试用例
stage-cli run-case \
  --case-file test-cases/005-投资项目新增-测试用例.md \
  --case-id INVEST-PROJECT-20260320-007 \
  --snippet-args '{"login":{"role":"建设单位"}}' \
  --headed

# 生成测试报告
stage-cli report --result-dir stage-cli-test/20260320164459
